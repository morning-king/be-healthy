## 3. 内容管理系统操作手册

### 3.1 架构变更
**新架构**: `ContentRepository` -> 读取 `assets/quotes.json` & `assets/poems.json` -> 写入 Room Database。
**优势**:
- **维护便捷**: 编辑 JSON 文件比修改 Kotlin 代码更安全、直观。
- **性能优化**: 避免了在 Kotlin 类中持有巨大的静态列表对象。
- **可扩展性**: 轻松支持数千条数据导入。

### 3.2 如何新增内容

1.  **定位文件**:
    - 名言: `app/src/main/assets/quotes.json`
    - 诗词: `app/src/main/assets/poems.json`

2.  **添加条目**:
    
    **在 quotes.json 中追加对象**:
    ```json
    {
      "content": "新的名言内容",
      "source": "作者/出处",
      "category": "分类",
      "tags": "tag1,tag2"
    }
    ```

    **在 poems.json 中追加对象**:
    ```json
    {
      "content": "诗词正文\n换行请用\\n",
      "author": "作者",
      "title": "标题",
      "dynasty": "朝代",
      "category": "分类",
      "tags": "tag1,tag2"
    }
    ```

### 3.3 数据库更新机制
- **智能增量更新**: 应用启动时，`ContentRepository` 会读取 JSON 文件，并与数据库中的内容进行比对（基于 `content` 字段去重）。
- **自动同步**: 任何在 JSON 中新增的条目，都会自动插入数据库。
- **注意事项**: 不要修改 JSON 中已存在条目的 `content` 字段，否则会被视为新条目重复插入。

### 3.4 内容审核标准
1.  **版权**: 确保引用内容为公有领域（Public Domain）或已获授权。
2.  **准确性**: 核对原文、作者和出处，避免张冠李戴。
3.  **格式**: JSON 格式必须合法，注意转义字符（如换行符 `\n`，双引号 `\"`）。
