# 项目规则

- **UI开发规范**：必须遵循 [UI组件开发与布局完整性规避法则](../../../docs/standards/ui_component_specification.md)，严禁非标准布局嵌套，确保核心组件稳定性。
- **主题开发规范**：新增主题必须在 `ThemeIcons.kt` 中实现绘制逻辑，严禁在 UI 组件中硬编码 SVG 或 PNG 资源。
- Lint 检查命令：`./gradlew lint`，仅在 git 提交前运行
- 完成任务前先进行代码分析，如果有问题进行修复，然后再编译构建，确保任务完成交付时：代码分析没任何问题、编译构建成功、运行成功
- 完成任务前先确保编译构建成功（必须通过）
- 任务里面只要修改了代码或者配置文件，在完成时一定要进行编译构建确保成功（必须通过）
